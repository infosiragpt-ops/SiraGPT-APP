# Python dependencies for the document-generation sandbox.
#
# The Node backend's `code-sandbox.js` runs short Python snippets via
# the host `python3` interpreter, so every package the doc-generator
# templates import (sgpt_docx, sgpt_xlsx, sgpt_pptx, sgpt_pdf,
# sgpt_math_omml) must be available globally on the deploy machine.
#
# Pin to install via:
#   pip3 install -r backend/requirements.txt
# or in CI with `--user` if the runner has no system-wide site-packages.
#
# Keep the list short and load-bearing; add a comment for every package
# explaining the document feature it backs.

# Office document toolkit (already a transitive dep of docxtpl, but
# pinning makes the doc-generator's "available libraries" preamble
# truthful).
